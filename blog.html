Discuss the scope of var,let, and const

var is a global scope. Variable declare with var are hoisted to the top of the containing function or global scope, which mean you can use them before they are declared. var can be reassigned and updated. Developer should not use the var anymore instead they can use let or const.

let has block-level scope, which means it is scoped to the nearest enclosing block. A block in JavaScript involves opening and closing curly braces. let variables that can be reassigned another value.

const: const also has block-level scope, similar to let. Variables declared with const cannot be reassigned. They are constant and must be assigned a value when declared

Tell us the use cases of null and undefined

null: null is an assigned value. It means nothing.

undefined: undefined means a variable has been declared but not defined yet.
